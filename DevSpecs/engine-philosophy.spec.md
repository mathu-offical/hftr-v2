# ENGINES

- the idea of the engine is that it ingests company philosophy, combines it with it's internally defined focus/scope + capital availability + exit target date, to run it's internal pipeline to ultimately create trading policies + real executions + runtime trend-linked market data


So an engine includes it's own trend modules, trade modules, and verification modules. 

it will require inputs from research/library AND live data modules, as well as from a funds source.

it will output action/execution monitoring, as well as have data outputs in form of trade policies and actual data dumps that can then be used for further processing or storage, as well as funds outputs.

all nodes included in an engine will allow connections to other outside nodes.



we need to continue to build out engines for all other specialties.
